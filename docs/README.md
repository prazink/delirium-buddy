# Delirium Buddy Documentation

This folder records the product, engineering, and delivery decisions for Delirium Buddy.

The goal is to keep a clear record of what was built in each phase, why each decision was made, and what comes next. This helps with product planning, technical interviews, portfolio explanation, and future SaaS upgrades.

## Current product direction

Delirium Buddy is a local-first Expo / React Native app for personal delirium-related change tracking, carer observations, and shareable care conversation summaries.

The app is intentionally positioned as personal tracking and care conversation support only. It does not diagnose, predict, treat, prevent, or replace medical assessment.

## Phase record rule

Every future phase should include:

1. Problem being solved
2. Scope included
3. Scope intentionally excluded
4. Technical changes
5. Product value
6. Safety/privacy notes
7. Manual QA checklist
8. Interview talking points

This keeps the project professional and easy to explain.
